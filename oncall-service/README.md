# On-call Readiness Service

Веб-сервис, переданный вам в сопровождение. Завтра он уходит в прод, а вы
заступаете на дежурство. Документация — то, что перед вами. Остальное —
ваша задача (см. текст лабораторной работы).

## Что это

Маленький CRUD-сервис заметок (`note`) на Python/Flask:

- хранит данные в **PostgreSQL**;
- кэширует чтение в **Redis** с TTL и **fallback в БД** при промахе или
  недоступности кэша;
- отдаёт **RED-метрики** в формате Prometheus на `/metrics`;
- имеет liveness/readiness-пробы для оркестратора.

## API

| Метод  | Путь            | Назначение                          |
|--------|-----------------|-------------------------------------|
| POST   | `/notes`        | создать заметку (`{title, body}`)   |
| GET    | `/notes`        | список заметок                      |
| GET    | `/notes/<id>`   | прочитать заметку                   |
| PUT    | `/notes/<id>`   | обновить (`{title?, body?}`)        |
| DELETE | `/notes/<id>`   | удалить                             |
| GET    | `/healthz`      | liveness                            |
| GET    | `/readyz`       | readiness (проверяет связь с БД)    |
| GET    | `/metrics`      | метрики Prometheus                  |

Ответ чтения содержит поле `source` (`cache` или `db`) — удобно для проверки
работы кэша и fallback.

## Конфигурация

Все параметры — через переменные окружения. Шаблон в `.env.example`:
хост/порт/креды БД и Redis, `REDIS_TTL`, `APP_PORT`, `LOG_DIR`.

Приложение логирует одновременно в файл (`$LOG_DIR/app.log`, с встроенной
ротацией) и в stdout.

## Запуск (для локальной проверки)

Требуются работающие PostgreSQL и Redis (как их разворачивать в целевой
среде — часть задания).

```bash
pip install -r app/requirements.txt
export $(grep -v '^#' .env | xargs)   # или задайте переменные иначе
python app/app.py                      # dev-режим
# прод-режим:
# gunicorn -w 4 -b 0.0.0.0:8080 --chdir app wsgi:app
```

Схема БД (`notes`) создаётся приложением при старте автоматически.

## Генерация трафика

`./loadgen.sh http://localhost:8080 1000` — создаёт заметку и шлёт N запросов
на чтение. Нужно, чтобы на дашбордах и в SLO-расчётах появились реальные
данные.

## Замечания

- Контейнеризация (Dockerfile), манифесты/чарт, мониторинг, алерты, бэкапы и
  автоматизация **намеренно отсутствуют** — это ваша работа.
